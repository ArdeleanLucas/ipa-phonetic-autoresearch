# ipa-phonetic-autoresearch

**Autonomous LLM-driven experimentation to hyper-specialize Wav2Vec2ForCTC models for precise IPA phoneme recognition of rare or dialect-specific patterns**

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)
[![Python 3.11+](https://img.shields.io/badge/Python-3.11%2B-blue)](https://www.python.org)

**Research Question**
Can Karpathy's AutoResearch loop discover reproducible, linguistically meaningful improvements in a Wav2Vec2 CTC model's ability to transcribe a *single target phonetic pattern* into accurate IPA symbols, using only 100–2000 human-transcribed audio clips?

Drop your audio + exact IPA transcripts into the repo, launch the loop overnight, and wake up to a git log containing **only validated improvements** on targeted Phoneme Error Rate (PER).

## Input Requirements (Read This First)

**This repo optimizes transcription quality against IPA labels you already have. It does not generate IPA for you.**

You must provide:
- `data/raw/*.wav` — audio clips of the target speech.
- `data/raw/*.ipa.txt` — **human-verified IPA transcriptions**, one space-separated phoneme sequence per line, matching each `.wav` file. Narrow transcription preferred when dialectal detail matters.

What this repo is **not**:
- ❌ **Not a speech-to-IPA transcription tool.** If you need raw IPA output from audio, use a pretrained phoneme recognizer (e.g. `facebook/wav2vec2-xlsr-53-espeak-cv-ft`, `facebook/wav2vec2-lv-60-espeak-cv-ft`, Allosaurus) *first* — then hand-correct the output before using it here.
- ❌ **Not an orthography-to-IPA converter for dialect research.** A `phonemizer`/eSpeak path exists for convenience, but it emits **citation-dialect IPA** and will silently erase the dialect-specific features this repo is designed to measure. Using it on dialect/clinical/L2 data makes the targeted-PER metric meaningless.
- ❌ **Not a pseudo-labeling pipeline.** Labels generated by another model will bias the loop toward that model's errors.

Ground truth must actually capture the phonetic pattern you want to improve on. If it doesn't, no amount of loop iteration will fix that — the metric you're optimizing is wrong.

See [`DATA_STATEMENT.md`](DATA_STATEMENT.md) for the annotation checklist.

## Features
- Native `Wav2Vec2ForCTC` + custom IPA tokenizer
- Targeted + weighted PER (articulatory-feature distance)
- Multiple loop variants (classic, config-driven, multi-agent)
- Full linguistic scaffolding (`DATA_STATEMENT.md`, `ETHICS.md`, qualitative notebook)
- Baselines, ablations, generalization tests, Praat export

## Quick Start
```bash
git clone https://github.com/ArdeleanLucas/ipa-phonetic-autoresearch.git
cd ipa-phonetic-autoresearch
uv sync
python prepare.py --help
python harness.py --model facebook/wav2vec2-base
```

## Limitations & Roadmap
- Best for modest datasets (100–2000 clips)
- Requires a capable LLM (Claude 3.5/4o or Grok recommended)
- Experimental — always validate final models with human phoneticians
